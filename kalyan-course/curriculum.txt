COURSE-LEVEL NOTES (for reference, not lecture content)
=====================================================================
Stack: Docker, Terraform, Kubernetes (EKS), Karpenter, HPA, Helm, ArgoCD,
GitHub Actions, OpenTelemetry/ADOT, AWS X-Ray, CloudWatch, AMP, AMG.

Reference project: 5-microservice retail store app
  - UI Service        (Java Spring Boot)
  - Catalog Service    (Go)               -> AWS RDS MySQL
  - Carts Service       (Java Spring Boot) -> AWS DynamoDB
  - Orders Service      (Java Spring Boot) -> AWS RDS PostgreSQL + AWS SQS
  - Checkout Service    (Node.js)          -> Amazon ElastiCache (Redis)

Key themes called out by the instructor across sections:
  - Cost optimization: ~70% savings via Karpenter Spot instances (Section 17)
  - Zero-downtime patterns: PodDisruptionBudgets + Spot interruption handling (Section 17)
  - OIDC-based CI auth: no static AWS access keys in GitHub Actions (Section 21)
  - Observability cost control: ~85% cost reduction by filtering health-check traces (Section 20)
  - GitOps: GitHub Actions (CI) -> ECR -> Helm values update -> ArgoCD (CD) auto-sync (Section 21)
